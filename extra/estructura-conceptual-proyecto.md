# Estructura conceptual del proyecto — Portafolio

**Visión general del proyecto · Elementos a tener en cuenta en todo el desarrollo de esta página web (portafolio).**

---

## Idea central

Cada **producto** que aparezca en el portafolio —ya sea una imagen, un logo, un video, una pieza de diseño o cualquier otro entregable— **va atado a un proceso**.

No se muestra solo el resultado final: se cuenta **cómo se llegó** a ese resultado.

---

## Contar dos historias

En el portafolio se presentan **dos niveles** para cada pieza:

1. **El proceso**  
   Las actividades, decisiones, aprendizajes y pasos que formaron parte de la creación (investigación, bocetos, pruebas, herramientas, errores y soluciones, etc.).

2. **El producto final**  
   El entregable terminado (el video, la imagen, el logo, etc.) tal como se presenta al espectador.

Objetivo: que quien recorra el portafolio entienda tanto **qué se hizo** como **cómo se hizo**.

---

## Implicaciones para el desarrollo del sitio

- Cada entrada o “ítem” del portafolio debería poder **enlazar o desplegar** tanto la pieza final como la documentación del proceso.
- La estructura de la web (páginas, secciones, navegación) debe permitir **asociar proceso ↔ producto** de forma clara.
- El contenido futuro (textos, medios, enlaces) se organiza con esta dualidad en mente: **proceso + producto**.

---

## Resumen en una frase

> **Cada producto del portafolio tiene un proceso; el portafolio cuenta las dos historias: la del proceso creativo y la del producto final.**
